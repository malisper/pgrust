## Current milestone

This branch makes the SQLancer `WHERE` oracle useful against pgrust by getting
past the first round of setup, syntax, scalar, and protocol blockers.

Fixed in pgrust:

- information schema table/column metadata for SQLancer schema loading
- bool-to-int casts generated by SQLancer
- all-NULL `IN` list typing
- `SELECT ALL`
- PostgreSQL legacy `FROM table*` inheritance marker
- `upper(text)`
- `BETWEEN SYMMETRIC`
- extended-protocol row descriptions for DML `RETURNING`

Fixed in the SQLancer checkout:

- allowlisted SQLancer-generated invalid comma/explicit-JOIN scope references
  when pgrust reports them as `column tN.cN does not exist`
