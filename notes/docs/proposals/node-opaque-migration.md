# Proposal: Opaque trait-object `Node<'mcx>` for pgrust-fabled

> Status: proposal (awaiting go-ahead on phasing). Goal: eliminate the `types-nodes` build
> blowup (16 MB rlib = 8.6 MB match-arm object + 5.7 MB metadata read by **445** dependent crates)
> by replacing the 241-variant `enum Node<'mcx>` with an opaque `PgBox<'mcx, dyn NodePayload<'mcx>>`,
> following the proven `src-idiomatic` generator design.

## 0. Executive summary

Replace the hand-written 241-variant `pub enum Node<'mcx>` (and the 49-variant `pub enum Expr`) in
`crates/types-nodes` with an opaque `PgBox<'mcx, dyn NodePayload<'mcx> + 'mcx>`, adapted to three
pgrust-fabled-specific realities `src-idiomatic` does NOT face:

1. **`Node` carries `'mcx`** (src-idiomatic's `Node` is `'static`), so `core::any::Any` is unavailable
   — we use a **tag-keyed unchecked downcast** (exactly C's `castNode`).
2. **`PgBox` = `allocator_api2::Box<T, Mcx>` on stable** — no `CoerceUnsized`, so unsizing to `dyn`
   is a **manual fat-pointer coercion** via `into_raw_with_allocator`/`from_raw_in`.
3. **`clone_in` is fallible and needs `mcx`** (`PgResult<Node<'b>>`) — the trait method threads the context.

Two decisive findings:
- **The enum is NOT generated by `build.rs`.** `build.rs` generates a separate 7-entry `node_tree::Node`;
  the real 241-variant `Node<'mcx>` is **hand-written** (`src/nodes.rs:328`), `Expr` at `primnodes.rs:1635`.
  So the generator must first *source* the variant table (parse the enum) — see Phase 1.
- **All 241 payloads are `crate::`-local** (zero external-crate payloads) — so the crate-split is an
  **intra-crate module extraction**, not a cross-crate move. Favorable.

## 1. Target representation

### 1.1 The trait
```rust
pub trait NodePayload<'mcx>: 'mcx {
    fn node_tag(&self) -> NodeTag;
    fn clone_in_dyn<'b>(&self, mcx: Mcx<'b>) -> PgResult<Node<'b>>;   // fallible + threads mcx
    fn equal_dyn(&self, other: &dyn NodePayload<'_>) -> bool;
    fn walk(&self, w: &mut dyn FnMut(&Node<'mcx>) -> bool) -> bool;    // hot path → vtable
    fn walk_mut(&mut self, w: &mut dyn FnMut(&mut Node<'mcx>) -> bool) -> bool;
    fn plan_base(&self) -> Option<&PlanBase<'mcx>> { None }           // (Plan*) upcast
    #[doc(hidden)] fn __payload_ptr(&self) -> *const () { (self as *const Self).cast() }
}
```
Lifetime-parameterized → cannot be `: 'static` → cannot require `Any`. `walk` folded into the vtable
because `expression_tree_walker` (the hottest loop) is today a 140-arm match — fabled-specific addition.

### 1.2 `Node` and `PgNodeBox`
A dedicated newtype `PgNodeBox` hides the verbose `PgBox<…, dyn …>` signature and owns the unsize
coercion (§1.4) + the tag-keyed downcast (§1.3) — **all the `unsafe` lives in this one type**:
```rust
pub struct PgNodeBox<'mcx>(PgBox<'mcx, dyn NodePayload<'mcx> + 'mcx>);  // the smart pointer
pub struct Node<'mcx>(pub(crate) PgNodeBox<'mcx>);                      // the node handle
```
`PgNodeBox::new(mcx, payload)` does the coercion; `PgNodeBox::downcast_ref::<Var>()` / the `Node::as_var`
accessors do the tag-keyed cast. The verbose `dyn NodePayload<'mcx> + 'mcx` type appears **nowhere else**
in the tree.

### 1.3 Downcasting — tag-keyed, NO `Any`
We do **not** use `core::any::Any` and do **not** need to. `Any` is `src-idiomatic`'s mechanism for
"given a `&dyn`, what concrete type is it?"; we have a better witness already — the `NodeTag` (C's
`castNode`). (`Any` is also unavailable anyway since it requires `'static` and our payloads carry `'mcx`,
but the point is we wouldn't use it regardless.) No `Any` supertrait, no `as_any()`, no `TypeId`. Each
adapter is `#[repr(transparent)]` over its payload, so the downcast is a tag check + a pointer cast:
```rust
pub fn as_var(&self) -> Option<&Var<'mcx>> {
    if self.0.node_tag() == ntag::T_Var {
        // SAFETY: T_Var is produced ONLY by NodePayload_Var (repr(transparent) over Var),
        // emitted from the same generator row → bijective. The box is + 'mcx so the borrow is sound.
        Some(unsafe { &*(self.0.__payload_ptr() as *const Var<'mcx>) })
    } else { None }
}
```
**Soundness:** (1) tag↔adapter bijection is generator-owned (a new tag-uniqueness assert guards it);
(2) `repr(transparent)` ⇒ same address; (3) lifetime is *recovered* through `&self` + the `+ 'mcx` box,
not invented. Strictly safer than `Any` (one transparent field, integer witness we control).

**`Expr` stays an enum for now** (only 49 arms; nested as the single `Node::Expr` variant; the blowup is
the 241-variant `Node`). Accessors route through it. Flipping `Expr` too is optional, deferred.

### 1.4 Constructors — the `mcx`-to-box divergence
src-idiomatic uses free `Box::new`; we need an `Mcx` to allocate. So `Node::Var(v)` → `Node::Var(mcx, v)`
(a heap alloc here *requires* a context — the mcx invariant). The name is preserved; the extra arg is a
mechanical, separate sweep. The unsize coercion (no `CoerceUnsized` on stable):
```rust
fn box_payload<'mcx, P: NodePayload<'mcx> + 'mcx>(mcx: Mcx<'mcx>, p: P)
    -> PgBox<'mcx, dyn NodePayload<'mcx> + 'mcx> {
    let sized = PgBox::new_in(p, mcx);
    let (raw, alloc) = PgBox::into_raw_with_allocator(sized);
    let raw: *mut (dyn NodePayload<'mcx> + 'mcx) = raw;   // *mut P -> *mut dyn IS legal on stable
    unsafe { PgBox::from_raw_in(raw, alloc) }
}
```
`new_in`/`into_raw_with_allocator`/`from_raw_in` all exist on stable allocator-api2 0.2.21. The raw
`*mut P -> *mut dyn` unsizing cast is available on stable (only the `Box: CoerceUnsized` impl is nightly).
This is the linchpin. Helper likely belongs in `mcx` as `PgBox::unsize_dyn`.

## 2. Generator changes (`crates/types-nodes/build.rs`)
Source the variant table by **parsing the hand-written enum** (Option A, ~80 LOC, recommended for P1/P2),
then mechanically emit the authoritative `nodes.list` at P3. `build.rs::emit` must produce: the trait;
`#[repr(transparent)]` adapters (mandatory for the downcast); the per-variant `impl NodePayload`
delegating to the existing `#[derive(PgNode)]` impls (`clone_in_dyn` threads mcx; `equal_dyn`/`walk` from
the lifted match arms); constructors; the full `as_/expect_/is_/into_` accessor set (tag-keyed cast bodies);
the complete `ntag::T_*` const module (the O(1) jump-table enabler); `copy/equal/out/read` free fns.
Plus a **tag-uniqueness assert**. ~400–500 LOC added; **2–3 days**.

## 4. Risk gates — measure before P3
1. **Hot-walker vtable benchmark (blocking).** `expression_tree_walker` is today a direct jump-table;
   trait `walk` makes each recursion an indirect call. Bench a large tree both ways; accept within ~10–15%,
   else keep a `node_tag()`-keyed match in the walker (the flip doesn't *require* `walk` in the vtable).
2. **Unsafe-downcast soundness (blocking).** Validated against rust-lang forum #75906 + the `better_any`
   crate: ours is a *safer* variant of that known pattern — it never round-trips through `'static` and never
   uses `Any` (the lifetime rides on `dyn NodePayload<'mcx> + 'mcx` throughout), so it avoids their worst
   hazards (transmute_copy double-borrow, `Drop`-observes-`'static`). The two load-bearing conditions both
   sources require:
   - **(a) Single lifetime per payload — BUILD-TIME ASSERT.** The downcast hardcodes `'mcx` as the *sole*
     lifetime; a `Foo<'mcx,'b>` payload would have no witness for `'b` and would fabricate `'b = 'mcx` →
     instant UB. The generator MUST assert every payload is `T<'mcx>` with exactly one lifetime (hard error
     otherwise). [= thread condition #2 / better_any's "single lifetime only".] The Phase-1 parser already
     enumerates all 296 payloads — add the check there.
   - **(b) Invariance in `'mcx` — COMPILE-FAIL TEST.** `dyn NodePayload<'mcx>` is invariant in `'mcx`
     automatically (trait-object params are invariant) — the thread's #1 fix, free. Add a compile-fail test
     that `Node<'long>` does NOT coerce to `Node<'short>`, guarding against a stray covariant
     `PhantomData<&'mcx ()>`. (Invariant where the current enum may be covariant — safety-preserving; expect
     minor compile friction at lifetime-shrinking sites; errors, never UB.)
   - Plus: tag↔adapter **bijection** (tag-uniqueness assert — our `TypeId`-uniqueness analog),
     `#[repr(transparent)]`, and **Miri** over `PgNodeBox`'s unsafe (+ `equal_dyn`). A tag collision or a
     2-lifetime payload is instant UB — highest severity. `better_any` is the audited fallback.
3. **rustc-ICE / lifetime-dyn spike.** Build a 1-variant `Node` end-to-end (construct/unsize/downcast/
   clone_in/walk/drop) on the pinned stable toolchain + Miri before P3. Fallback if the manual coercion
   fails: enum-of-pointers thin tag+union (heavier, deferred).
4. **Metadata-size regression check.** After P3, re-measure rlib object+metadata + a warm fan-in rebuild
   before investing in P4.

## 6. Ground-truth ctor-site audit + concrete atomic-flip plan (2026-06-19, node-refactor lane)

This section records the measured construction-site landscape and the **exact** atomic-flip
(P3) recipe, so the flip can be done as one deliberate reviewed campaign.

### 6.1 The constructor layer already exists and is swappable
`build.rs::emit_constructors` (lib.rs `include!`s `$OUT_DIR/node_constructors.rs`) already emits a
`Node::mk_<snake_variant>(mcx, payload) -> Node<'mcx>` for every Node-direct variant **and** a
`Node::mk_<exprvariant>(mcx, payload)` for every Expr leaf (routed through `Node::Expr(Expr::V(..))`),
plus the whole-Expr `mk_expr(mcx, Expr)`. **Today every `mk_*` body is byte-for-byte the matching
`Node::Variant(payload)` enum construction (mcx ignored: `_mcx`).** At the flip ONLY these bodies
change — to `PgNodeBox::new(mcx, NodePayload_<V>(payload))`. So routing all direct construction through
`mk_*` makes the representation swappable behind a single generated file. ~976 sites already route
through `mk_*`; this lane migrated 48 more (commits below).

### 6.2 The TWO separate Node enums — do NOT conflate
`types_nodes::Node` (251 variants, **the flip target**, imported by 275 crates) is distinct from
`types_parsenodes::Node` (a separate ~18-variant raw-grammar value enum, imported by 39 crates) and from
the family enums `PathNode` / `PlanNode` etc. A naive `grep 'Node::'` triple-counts. The *target*
construction sites (resolving which `Node` is in scope per-file + excluding `*Node::` glued idents and
fully-qualified `types_parsenodes::Node::`) number **~219 outside types-nodes + ~259 inside types-nodes**.

### 6.3 Remaining target-Node construction sites by FRICTION CLASS (~161 non-comment, excl types-nodes)

which require the same structural change the flip performs. **Correction (node-refactor lane 2026-06-19):**
the `makeInteger`/`makeFloat`/`makeString`/`makeBoolean`/`makeBitString` value helpers in
`backend-nodes-core/value_core.rs` build **`types_parsenodes::Node`** (the separate raw-grammar value
enum), **NOT** the flip-target `types_nodes::Node<'mcx>` — so they are NOT in any flip friction class and
need no `mcx`. The original §6.3 table conflated the two enums for that row. Updated counts:

### 6.4 The invariance-friction finding (validates §4.2b empirically)
The today-enum `Node<'mcx>` is **covariant** in `'mcx` (it is a plain enum of payloads). Many in-place
mutators exploit this: e.g. `backend-optimizer-util-vars/flatten.rs` deliberately writes lifetime-free
`Expr` values into a `&mut Node<'_>` "at any lifetime" (its own module doc says so), and
`node_walker`'s `expression_tree_walker_mut` closure receives the node at a **higher-ranked** lifetime.
Routing those through `mk_expr(mcx, ..)` produces a `Node<'mcx>` whose lifetime is pinned to the local
`mcx`, which no longer coerces into the param's shorter/HRTB lifetime → **E0621 / "lifetime may not live
long enough"**. This is exactly the predicted P3 invariance friction (the opaque `dyn NodePayload<'mcx>`
is invariant), surfacing *early* at these mutator sites. **Fix is the same in both worlds:** thread an
explicit `'mcx` onto the `&mut Node<'mcx>` mutator params (and their walker-closure HRTB). That is a P3
prerequisite, NOT a behavior-preserving pre-batch — it must land **with** the flip (or as a tightly-scoped
predecessor) because pinning the param lifetime before the rep changes can break covariant callers.

### 6.5 Concrete atomic-flip (P3) recipe — single reviewed campaign
Prereqs (land first, independently, behavior-preserving): finish migrating the **~61 `value_with_mcx`**
sites to `mk_*` (this lane did the isolated ones); they shrink the flip's "constructor-form" delta toward
the irreducible `value_no_mcx` + mutator classes.

The flip commit itself:
1. **Codegen (`build.rs`):** add the trait + `#[repr(transparent)]` per-variant `NodePayload_<V>`
   adapters + `PgNodeBox`/`Node` newtypes + the tag-keyed `as_/expect_/into_` bodies (already emitted as
   enum-matches — switch them to the pointer-cast form) + the **tag-uniqueness assert** and the
   **single-`'mcx`-lifetime-per-payload assert** (§4.2a). Flip the ~5 `mk_*` body templates from
   `Node::Variant(p)` → `PgNodeBox::new(mcx, NodePayload_<V>(p))`. The `mk_expr` body boxes a
   `NodePayload_Expr` over the still-enum `Expr` (Expr flip deferred, §1.3).
2. **Enum swap:** delete the hand-written `pub enum Node<'mcx>` (nodes.rs:328) + its hand `node_tag()` /
   `clone_in()` / `Display`; `Node` becomes the `PgNodeBox` newtype. `mcx::PgBox::unsize_dyn` helper
   (§1.4) lands in `crates/mcx`.
3. **Constructor sweep (rustc-guided):** every remaining direct `Node::Variant(p)` becomes
   `Node::mk_variant(mcx, p)`. The **`value_no_mcx` (~121)** sites get `mcx` threaded into their fn
   signature here (rustc points at each); the `make_*` value helpers in `value_core.rs` gain an `mcx`
   param (and ~all their callers) — this is the single largest mechanical sub-sweep.
4. **Mutator lifetime sweep (§6.4):** thread `'mcx` onto the `&mut Node<'mcx>` params of the in-place
   mutators (`flatten.rs`, `node_walker.rs` `*_mut` closures, `rewrite-core/replace.rs`,
   `parser-agg`) and their HRTB walker closures. ~37 sites + their call chains.
5. **Match sites:** P2 already converted structural `Node::Variant(x) =>` matches to
   `node_tag()`/`expect_*`. Audit for stragglers (the ~66 `pattern` hits above + any `matches!`) — each
   must be `expect_*`/`as_*` before the enum is gone. **This is the true P3 gate: zero structural matches
   on `Node` may remain.**

### 6.6 Blast radius + ordering
- **Blast radius:** the enum-swap recompiles all **275** `types_nodes::Node` consumers (the metadata
  shrink is the payoff; full fan-in dissolution waits for P4 crate-split). The constructor + mutator
  sweeps touch ~40 crates (the per-crate counts in this lane's audit). The `value_no_mcx` `mcx`-threading
  is the riskiest (signature changes ripple through call chains), so stage it as its own pre-flip lane
  where possible by adding the `mcx` param **while still on the enum** (behavior-preserving: the param is
  `_mcx`-unused until the flip).
- **Ordering:** (a) finish `value_with_mcx`→`mk_*` [behavior-preserving, parallel]; (b) thread `mcx` into
  `value_no_mcx` fn signatures while still on the enum [behavior-preserving]; (c) ensure zero structural
  `Node` matches remain [P2 closeout]; (d) the atomic flip = codegen + enum-swap + mutator-`'mcx` sweep,
  one commit, rustc-guided. Steps (a)/(b)/(c) are independent green pre-batches; (d) is the single
  reviewed campaign behind the §4 risk gates (walker vtable bench, Miri over `PgNodeBox`, 1-variant spike).
